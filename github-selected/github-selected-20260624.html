chopratejas / headroom

LLM 上下文压缩层：在内容进入模型前削减 60%–95% 的 Token

TOP 1
上下文压缩
Repository: chopratejas/headroom语言：Python / TypeScript主题：LLM · Token 优化 · MCP

Headroom 是一个「上下文优化层」，定位是夹在你的应用和 LLM 之间，对所有进入上下文窗口的内容——工具输出、日志、文件、RAG 片段、对话历史——做智能压缩，再把更精简的 prompt 转发给模型。它的卖点是「同样的答案，更少的 Token」，官方 README 给出的实测包括代码搜索约省 92%、SRE 排障约省 92%、GitHub Issue 分类约省 73%。它提供三种集成方式：作为库直接调用、作为代理透明拦截、或作为 MCP Server 接入。和简单的「截断 / 摘要」不同，它强调不丢关键信息（比如日志里那行真正解释崩溃的报错）。

技术亮点

  • 三种集成模式：Library、Proxy、MCP Server 任选，无论你怎么调 LLM 都能接入。
  • 面向 Agent 场景：专门压缩工具输出和 JSON 这类高冗余内容，而非粗暴砍掉中间段落。
  • 跨语言可用：提供 headroom-ai 的 Python 与 Node 包，以及官方 Docker 镜像。
  • 可自学习：能分析历史会话、找出失败的工具调用并把纠正写回 CLAUDE.md / MEMORY.md，让下次会话更聪明。

使用场景

  • 降本增效：为高频调用 LLM 的应用直接削减 Token 账单与延迟。
  • Agent 工具链：代码搜索、日志排障、Issue 分类等工具输出量大的环节最受益。
  • RAG 系统：在检索片段进入 prompt 前压缩，腾出更多有效上下文。
  • 本地代理：用 Proxy 模式给现有 Claude Code / Codex 工作流「无侵入」加压缩。

openclaw / openclaw

跑在你自己设备上的个人 AI 助手 🦞

TOP 2
本地优先助手
Repository: openclaw/openclaw主题：Personal AI · Local-First · Agent理念：数据主权

OpenClaw 的口号是「你自己的个人 AI 助手，任何系统、任何平台」。它是一个本地优先（Local-First）的自主 AI Agent：跑在你自己的 Mac / Windows / Linux 上，通过你已经在用的聊天工具（WhatsApp、Telegram、Slack、Discord、飞书、微信、iMessage 等）响应你，并能自主完成浏览器操作、文件管理、收发邮件、日历管理、设置提醒等任务。它的核心理念是「Gateway 只是控制平面，真正的产品是这个助手」，强调上下文、个人偏好和对话历史都留在你自己的硬件上，而非科技巨头的服务器。它也是近期开源 AI 圈增长极快的项目之一。

技术亮点

  • 本地优先：数据和处理默认留在本机，隐私与数据主权是第一设计目标。
  • 多渠道接入：把已有的即时通讯工具当作交互入口，无需再装一个新 App。
  • 自主执行：不只是问答，能控制浏览器、读写文件、执行 shell、处理邮件日历。
  • 跨平台可控：支持 macOS / iOS / Android 语音交互，并可渲染由你掌控的实时 Canvas。

使用场景

  • 私人数字管家：用熟悉的聊天窗口统一处理消息、提醒、邮件和待办。
  • 本地自动化：把日常的浏览器操作、文件整理、信息汇总交给常驻 Agent。
  • 隐私敏感场景：不希望个人数据上云的用户，自建可控的 AI 基础设施。
  • Agent 学习样本：研究本地优先架构、多渠道接入与自主任务执行的工程实现。
