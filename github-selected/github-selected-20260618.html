1. openai/codex

运行在终端里的轻量级 AI 编码 Agent，将自然语言需求转化为可执行的工程变更

TOP 1
🤖 Coding Agent🦀 Rust / TypeScript🧰 CLI / SDK🏷️ terminal / code review / automation

项目简介：Codex 是 OpenAI 官方开源的轻量级编码 Agent，定位是“runs in your terminal”。它面向真实代码仓库工作，可以通过命令行安装和使用，并提供 CLI、IDE、桌面体验以及 SDK 等不同入口。相比只在编辑器里补全代码，Codex 更像一个可以接收任务、阅读上下文、修改文件并协助检查代码的工程执行助手。

技术亮点

  • 终端优先：直接在开发者熟悉的 shell 环境中运行，适合和现有 git、测试、构建流程串联。
  • 多入口形态：除了 CLI，还能接入 IDE、桌面应用和 SDK，覆盖个人使用、插件化集成与自动化调用。
  • 工程任务闭环：面向代码生成、编辑、审查与迭代，不只是问答，而是围绕仓库执行连续任务。
  • 可编程能力：TypeScript / Python SDK 让团队可以把 Codex 接入内部研发工具、质量门禁或自动化流程。

使用场景

  • 日常开发提效：让 Agent 处理小型重构、测试补齐、脚手架生成和问题定位。
  • 提交前自检：借助独立 Agent 做代码审查，提前发现遗漏、边界条件或风格问题。
  • 自动化研发助手：把重复性工程任务包装成命令或脚本，让 Codex 在标准流程中执行。
  • AI 编码工具评估：适合团队验证终端 Agent 与现有 IDE 插件、Web Agent 的效率差异。

为什么今天推荐它

  • AI 编码工具正在从“聊天式辅助”转向“任务式执行”，Codex 是观察这条路线的核心项目之一。
  • 它由 OpenAI 官方维护，文档与生态入口相对完整，适合作为团队评估 AI 编码工作流的基准样本。
  • 对开发者个人而言，它能直接进入日常终端工作流，学习成本和落地成本都比较低。

2. x1xhlol/system-prompts-and-models-of-ai-tools

收集主流 AI 工具系统提示词、内部工具描述与模型配置的逆向学习资料库

TOP 2
🧠 Prompt Engineering📚 AI Tools📄 GPL-3.0🏷️ Cursor / Claude Code / Devin / Replit / v0

项目简介：这个仓库汇集了作者整理的多种主流 AI 工具系统提示词、内部工具说明和模型相关信息，覆盖 Cursor、Claude Code、Devin、Replit、Windsurf、VSCode Agent、v0 等一批开发者熟悉的产品。它的价值不在于“复制提示词”，而在于帮助开发者观察成熟 AI 产品如何定义角色、约束行为、暴露工具、处理上下文与规划任务。

技术亮点

  • 横向样本丰富：同一仓库里能对比多类 AI 编程工具、Agent 产品和生成式应用的系统设计差异。
  • 关注工具调用：不仅有提示词文本，还包含内部工具描述，适合研究 Agent 如何把语言模型连接到外部能力。
  • 产品机制拆解：可观察不同产品如何设置安全边界、任务规划、上下文管理和输出规范。
  • Prompt 工程素材库：适合提炼通用模式，再改造为企业内部助手、研发 Agent 或自动化工作流。

使用场景

  • 学习 Agent 设计：分析成熟工具的角色定义、工具 schema、任务分解和错误处理策略。
  • 内部工具建设：为企业知识库助手、代码助手、运营助手设计系统提示词提供参考。
  • 竞品研究：对比 Cursor、Claude Code、Windsurf 等产品的交互边界和能力暴露方式。
  • 提示词审计：帮助团队建立对 prompt 泄露风险、权限边界和安全策略的敏感度。

为什么今天推荐它

  • 当 AI 工具越来越像“可执行系统”而不是“聊天框”，系统提示词和工具协议就成为核心工程资产。
  • 这个项目能帮助开发者从产品表象深入到底层设计，理解好用的 Agent 为什么好用。
  • 对想做 AI 工具、内部 Agent 或 prompt governance 的团队，它是一个很适合拆解的参考样本。
