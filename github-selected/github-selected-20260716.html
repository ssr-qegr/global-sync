1. alibaba/page-agent：把网页直接变成可执行的 GUI Agent

page-agent 是阿里开源的页面内嵌式 GUI Agent，核心定位非常清晰：用纯 JavaScript 方式，把自然语言操作能力直接塞进网页本身。它不要求额外的 Python 服务、无头浏览器或复杂后端改造，而是通过文本化 DOM 操作来理解并执行页面交互。这种思路很适合需要快速给 SaaS、管理后台、表单系统和内部平台加“AI 副驾驶”的团队。

项目地址alibaba/page-agent
核心形态In-page GUI Agent / TypeScript
当前热度26.7k⭐ / 2.4k Fork（2026-07-16）
为什么今天选它：很多“AI + 浏览器”的方案仍然依赖截图、多模态或外部自动化框架，而 page-agent 直接把能力嵌进前端页面，工程路径更轻，也更适合产品化。
项目详细介绍

从官方 README 看，page-agent 提供一行脚本接入方式，也支持 npm 安装后以编程方式初始化。它支持自备 LLM，可以接入主流模型，甚至本地部署模型；还提供可选 Chrome 扩展用于跨页面任务，以及 MCP Server（Beta）用于让外部 Agent 客户端获得浏览器控制能力。换句话说，它既能作为单页面 AI Copilot 使用，也可以作为更大 Agent 系统里的“浏览器执行层”。

技术亮点
  • 纯页面内 JavaScript 集成，无需单独部署浏览器自动化后端，接入门槛低。
  • 基于文本的 DOM 操作，而非截图驱动，避免多模态成本和权限复杂度。
  • 支持自带模型、Chrome 扩展、多页面任务与 MCP，对接现有 Agent 体系很自然。
  • 很适合学习“产品内 AI 操作层”该如何设计，包括动作抽象、交互反馈与执行边界控制。
使用场景

适合做企业后台智能表单填写、SaaS 产品 AI 助手、无障碍增强、跨标签页网页操作，以及把自然语言指令映射成网页动作的智能前端实验。对技术成长来说，这个项目特别适合前端工程师、全栈工程师和做 Agent 产品的人深入研究：它展示了如何在真实 Web 环境中把“会聊天”升级成“会操作”。

2. opendatalab/MinerU：把复杂文档转成 LLM 可用的结构化知识

MinerU 是 OpenDataLab 开源的文档解析工具，目标非常明确：把 PDF、图片、DOCX、PPTX、XLSX 等复杂文档，转换成适合下游检索、抽取和 Agent 工作流消费的 Markdown 与 JSON。对于做知识库、RAG、文档问答、企业资料理解的人来说，这种“文档清洗 + 结构还原”能力是非常关键的底层基础设施。

项目地址opendatalab/MinerU
核心形态Document Parser / Python
当前热度74.7k⭐ / 6.2k Fork（2026-07-16）
为什么今天选它：很多团队在做 RAG 时真正的瓶颈并不是向量库，而是上游文档解析质量。MinerU 正好命中这个核心难点，而且能力边界非常完整。
项目详细介绍

官方说明里，MinerU 不只是“抽文本”，而是尽量保留原始文档结构，包括标题、段落、列表、图像、表格、脚注与公式等元素，同时支持 OCR、复杂布局、多语言识别和多种输出格式。它内置 CLI、FastAPI 和 Gradio WebUI，既能单机本地跑，也能扩展到多服务部署；对生产环境来说，这种“可工具化、可服务化”的设计很有价值。

技术亮点
  • 输入覆盖非常全：支持 PDF、图片、DOCX、PPTX、XLSX 等主流文档格式。
  • 输出面向 LLM 工作流：可生成 Markdown、JSON 与更丰富的中间格式，方便 RAG 和数据清洗。
  • 兼顾结构与语义：支持复杂版面、表格转 HTML、公式转 LaTeX、扫描件 OCR 与 109 种语言识别。
  • 工程落地友好：提供 CLI、REST API、Gradio WebUI，支持 CPU 运行，也支持 GPU / Apple Silicon 加速。
使用场景

适合做企业知识库预处理、论文与技术文档抽取、财报与报告解析、RAG 数据管道、Agent 文档阅读前处理，以及多模态资料结构化清洗。对技术成长来说，MinerU 很值得做深度学习：它能帮助你理解“高质量文档解析”在 AI 系统里到底有多重要，也能看到从算法能力到 API/CLI/部署形态的完整工程化路径。
