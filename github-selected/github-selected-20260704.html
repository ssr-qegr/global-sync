1. obra/superpowers

把"让 AI 写代码"变成一门有工程铁律的手艺——一个开源的 Agentic Skills 框架

TOP 1
⭐ 210,000+ Stars🍴 Fork 活跃💻 TypeScript / Markdown🪪 MIT🧠 Agentic Skills Framework

项目详细介绍：Superpowers 是由资深工程师 Jesse Vincent（obra）主导的开源项目，被社区称为"AI 编码 Agent 的工程铁律"。它并不训练一个新模型，也不做 IDE 插件，而是给 Claude Code、Cursor、Codex、Gemini CLI 这类"能真正写代码的 AI Agent"提供一整套 可加载、可组合、可审计 的 Skills 与工作流模板。核心是一个 7 阶段结构化流程：需求澄清 → 计划审批 → TDD 实现 → 子 Agent 并行 → 代码审查 → 交付收尾 → 复盘沉淀。每一阶段都有对应的 Skill Markdown 定义清楚"AI 应该做什么、必须先做什么、不允许做什么"，从而把常见的"Agent 一顿乱写、跑不过、瞎改再跑"的失败模式，改造成"先审批计划再写代码、先补测试再改实现、并行开子 Agent 处理独立任务、最后统一审查合并"的工程化流程。项目在 2026 年 6~7 月冲上 GitHub 全球 Trending 榜首，被多个团队作为"AI 结对编码规范"直接接入内部研发流程。

技术亮点

  • 7 阶段结构化工作流：把 Agent 编码从"聊天式提示词"升级为"有阶段、有产物、有审批"的工程流程。
  • Skills 可组合：每个 Skill 是一份 Markdown + 结构化指令，Agent 按需加载，避免上下文膨胀。
  • TDD-First 强约束：要求先写/生成测试，再让 Agent 实现，从工作流层面减少"能跑就行"的低质量代码。
  • Sub-Agent 并行：父 Agent 拆分独立子任务下发给子 Agent 并行执行，加速大改造类任务。
  • 与主流 Agent 无缝对接：可直接嵌入 Claude Code / Cursor / Codex / Gemini CLI，非侵入式扩展。

使用场景

  • 团队 AI 结对编码规范：把 Superpowers 作为团队内部"AI 使用手册"，统一每个人调用 Agent 的方式。
  • 大型重构 / 迁移任务：用子 Agent 并行改造多个模块，父 Agent 汇总审查，适合升级依赖、跨语言迁移。
  • 个人生产力提升：个人开发者用它约束自己的 AI，减少"改一处坏一处"的返工。
  • AI 教学 / 布道：作为"AI 编码最佳实践"课程材料，比抽象讲 prompt engineering 更实操。

为什么今天推荐它

  • 它给"AI 写代码"这件事补上了最缺的一块——工程规范。会用 AI 是 60 分，能让 AI 稳定产出高质量代码是 90 分，而 Superpowers 就是 60 → 90 的模板。
  • Skills 是纯 Markdown，可读性极高，即使不接入 Agent，只当"AI 编码守则"读一遍也很有收获。
  • 它验证了"结构化工作流 > 更强模型"这条 2026 年的核心工程认知：模型再强，也需要工程流程去约束和放大。

2. karpathy/nanochat

Andrej Karpathy 的"最小可跑 ChatGPT"——从 tokenizer 到 RLHF，一份代码全流程

TOP 2
⭐ 高星教育项目🍴 Fork 火爆💻 Python / PyTorch🪪 MIT🎓 LLM From Scratch

项目详细介绍：nanochat 是 OpenAI 联合创始人、深度学习教育界"祖师爷" Andrej Karpathy 在 nanoGPT 之后的进阶作品。如果说 nanoGPT 教你"如何从零训一个 GPT"，那么 nanochat 就是教你如何从零训一个能真正对话的 ChatGPT。它是一个可读性极高、单机就能跑的完整流水线，用一份精简 Python 代码串起了大模型训练的全部核心阶段：数据准备 → BPE Tokenizer → Pretraining → SFT（监督微调）→ RLHF / DPO 偏好训练 → 推理与聊天。项目提供交互式在线 Demo（nanochat.karpathy.ai），让你直接和自己训出来的迷你 ChatGPT 对话。README 与代码里 Karpathy 一以贯之的极简教学风格：注释说人话、代码不炫技、每一步都能被 print / debug 出来。整个训练可在一台带 GPU 的开发机上完成，估算成本约 100 美元级别，非常适合作为"读一遍就能真正理解 LLM"的教学项目。

技术亮点

  • ChatGPT 全流程复现：Pretraining + SFT + 偏好训练 + 推理，一份代码打通"从随机权重到能对话"。
  • 单机可跑 + 成本可控：单 GPU 训练，估算约 100 美元级预算，普通开发者可以真的做一遍。
  • 极简教育风格：Karpathy 系列一贯的"能读懂 > 能炫技"，每一步都能作为教学材料。
  • PyTorch 原生实现：不依赖 transformers / trl 等重型封装，直接暴露训练细节，方便逐行研读。
  • 在线交互 Demo：提供 nanochat.karpathy.ai，直接和自己训出来的模型对话，学习闭环。

使用场景

  • 系统学习 LLM 工程：研究生 / 转型 ML 工程师用来把"论文里的公式"变成"能跑的代码"。
  • 企业内训 / 团队 Onboarding：作为算法团队新人的入门项目，一周内做完，理解从 tokenizer 到 RLHF 的每一步。
  • 验证训练想法：在小规模上快速验证新的 loss / 采样 / 奖励函数是否有效，再放到大模型上。
  • 技术布道内容源：写博客 / 做 talk 时，用 nanochat 作为可视化例子讲清 LLM 内部机制。

为什么今天推荐它

  • 它是 2026 年"最有学习性价比"的 LLM 项目：投入几天时间 + 一张显卡，就能跑完 ChatGPT 类模型的全部关键阶段。
  • Karpathy 出品意味着代码质量、教学清晰度、社区讨论质量都有保障，遇到问题也基本都能搜到答案。
  • 作为工程师，读透 nanochat 之后，再去看 vLLM / DeepSpeed / TRL 这些工业级项目，会有"从"知道名词"到"真正懂原理""的跃迁。
