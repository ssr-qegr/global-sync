1. anomalyco/opencode

开源 AI 编码 Agent，直接运行在终端里，覆盖计划、开发、子任务搜索与桌面端体验

TOP 1
⭐ 179,845 Stars💻 TypeScript🧰 CLI + Desktop🤖 build / plan / subagent🏷️ coding agent / terminal / automation

项目详细介绍：OpenCode 的 README 直接把它定义为 “The open source AI coding agent.”，核心定位是让开发者在自己熟悉的终端环境中使用 AI 完成真实工程任务。它既支持一键安装，也提供桌面版，并内置 build、plan 两类 agent 以及 general subagent：build 负责高权限开发执行，plan 适合只读分析与代码探索，general subagent 用于复杂搜索和多步骤任务。这意味着它不只是一个“能补全代码的聊天框”，而是开始向“能接任务、看仓库、拆步骤、再落地执行”的工程助手演进。

技术亮点

  • 终端优先，贴近真实工作流：直接跑在 shell 里，天然适合接 git、测试、构建、脚本等已有研发流程。
  • 内置多角色 Agent：build 负责执行，plan 负责只读分析，适合在探索代码与真正改代码之间做职责分层。
  • 支持子代理能力：README 明确提到 general subagent，可把复杂搜索与多步任务拆出去，适合处理更长链路的问题。
  • 入口丰富：除 CLI 外还提供桌面应用，降低新用户上手门槛，也方便团队内部演示与推广。

使用场景

  • 日常开发提效：让它处理脚手架生成、批量重构、测试补齐、问题排查等重复性任务。
  • 陌生仓库分析：先用只读 plan agent 探索代码结构，再决定是否交给 build agent 真正落地改动。
  • 命令行驱动研发：适合偏 terminal 流派的工程师，把 AI 深度嵌进自己的本地开发习惯。
  • 团队 AI 编码试验田：可用于验证“终端 Agent”相对 IDE 插件或 Web 端 AI 工具的效率差异。

为什么今天推荐它

  • 它代表了一类很值得关注的趋势：AI 编码工具正在从“代码补全”走向“任务执行器”。
  • 终端形态对工程师非常自然，迁移成本低，容易快速进入真实生产流。
  • 如果你想建立自己的 AI 开发工作流，OpenCode 很适合作为基线项目来观察和学习。

2. nanobrowser/nanobrowser

开源浏览器 AI 自动化扩展，在本地浏览器里运行多智能体工作流，替代高门槛云端 Web Agent

TOP 2
⭐ 13,364 Stars🧩 TypeScript🌐 Chrome / Edge Extension🧠 Planner + Navigator🏷️ browser automation / multi-agent / local-first

项目详细介绍：Nanobrowser 是一个开源的 AI Web Automation 工具，直接以浏览器扩展的方式运行在本地浏览器中。README 将它定义为 OpenAI Operator 的免费替代方案，强调“用自己的 LLM API key”“本地运行”“多智能体协作”。它把浏览器场景里的任务拆成 Planner、Navigator 等不同角色，让一个 agent 负责规划，一个 agent 负责在网页中执行动作。这种设计很适合帮助开发者理解：Web Agent 要想真正可用，不只是模型强，还要把规划、导航、状态反馈和上下文追问一起做好。

技术亮点

  • 浏览器内本地运行：所有动作都在本地浏览器发生，隐私边界更清晰，账号凭证不必交给远端托管服务。
  • 多智能体协作：README 明确采用 Planner 与 Navigator 分工，体现了 Web Agent 常见的“规划-执行”架构。
  • 模型接入灵活：支持 OpenAI、Anthropic、Gemini、Ollama 以及兼容 OpenAI 协议的提供方，方便按成本与效果做组合。
  • 产品化完成度高：既能从 Chrome Web Store 安装，也支持手动加载最新版本，对普通用户更友好。

使用场景

  • 网页信息采集：比如抓取新闻站点头条、整理 GitHub 趋势仓库、汇总商品信息等。
  • 重复网页操作自动化：适合处理表单填写、后台巡检、信息录入、页面检索等高频重复任务。
  • Agent 产品研究：非常适合拿来观察 Web Agent 的任务规划、失败纠偏和人机交互设计。
  • 低成本搭建个人助手：如果你想试验浏览器自动化但不想依赖高价 SaaS，这个项目很有实践价值。

为什么今天推荐它

  • 浏览器 Agent 是 AI 落地里非常有想象空间的一条路线，而 Nanobrowser 的门槛相对更低。
  • 它把“本地运行、多模型接入、浏览器自动化”几个关键点组合得很完整，适合学习也适合直接试用。
  • 对想做自动化、副项目、效率工具的人来说，这类项目往往能直接启发产品原型。
