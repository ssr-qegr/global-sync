1. browser-use

让 AI Agent 真正“看网页、点按钮、跑流程”的浏览器自动化基础设施

TOP 1
⭐ 98,853 Stars 🍴 11,034 Forks 🐍 Python 📄 MIT License 🏷️ browser automation / llm / playright-like agent

项目介绍：browser-use 的定位很明确——把网站变成 AI Agent 可操作的真实环境。它不只是“开个浏览器脚本”，而是把自然语言任务、模型推理、浏览器动作空间和恢复机制串成一套完整运行链路，适合做网页检索、表单填写、流程执行、自动化测试和多站点信息采集。

技术亮点

  • Rust Core + Browser Harness：README 中明确提到新 beta agent 由 Rust 内核驱动，外层接浏览器执行环境，强调真实浏览器动作空间与恢复循环。
  • 对主流模型友好：支持通过 ChatBrowserUse 接入多种 provider，也支持和 Cursor、Claude Code 等编码 Agent 配合。
  • CLI / Python 双入口：既能直接写 Python Agent，也能通过 CLI 维持持久浏览器会话，适合快速试验和工程落地。
  • 生产化要素更完整：文档里强调云端浏览器、代理轮转、Captcha 处理、并发执行和持久化能力，说明它不是单纯 Demo 型项目。

使用场景

  • Agent 执行网页任务：登录站点、跨页面点击、下载文件、执行完整业务流程。
  • 信息采集与研究：从多个网站抓取结果后自动整理和总结，适合做监控、调研与日报。
  • 网页测试与回归：用自然语言描述测试步骤，驱动真实浏览器完成验证。
  • 个人效率助手：自动填表、投递申请、购物比价、帮你完成重复线上操作。

为什么今天推荐它

  • Agent 从“会聊天”走向“会干活”，浏览器是最直接也最通用的执行环境。
  • 它同时覆盖开源库、CLI、文档和云能力，说明生态已经比较成熟。
  • 如果你想把 AI 和真实业务网页流程结合，这是非常值得跟进的一类基础设施项目。

2. ai-engineering-from-scratch

从数学、训练、RAG、Agent 到安全评估的超大规模 AI 工程实战课程仓库

TOP 2
⭐ 32,521 Stars 🍴 5,341 Forks 🐍 Python 为主 📄 MIT License 🏷️ ai-engineering / agents / mcp / tutorial

项目介绍：这是一个把“AI 工程”当成完整学科来组织的仓库，而不是零散教程集合。项目官网和 README 给出的信息都很强：覆盖 20 个阶段、503 节课程，强调“自己写出反向传播、Tokenizer、Attention、Agent Loop”，让你真正理解模型与工程系统是怎么搭起来的。

技术亮点

  • 覆盖范围广：从数学基础、深度学习、LLM、RAG 到 Agent Engineering、Safety Harness、分布式训练，路径非常完整。
  • 不是只讲概念：README 强调每课都会产出可复用 artifact，最终沉淀 prompts、skills、agent workbench 等工程资产。
  • 工程化组织清晰：支持按 phase 安装技能、生成 catalog、运行 lesson audit、做脚手架，适合边学边建设自己的 AI 工程工具箱。
  • 面向 AI Coding Agent 生态：项目内有大量 SKILL、prompt、AGENTS 相关产物，对现在的 Agent 开发方式很贴近。

使用场景

  • 系统补课：适合想从“会调 API”升级到“懂训练、懂推理、懂 Agent 工程”的开发者。
  • 搭建团队学习路线：阶段化组织天然适合做内部训练营或个人长期学习计划。
  • 做 AI 项目脚手架参考：可以借鉴它对技能、课程、脚本和工程资产的组织方式。
  • 打造作品集：里面大量 capstone 和 end-to-end 项目，适合做实战输出和积累 demo。

为什么今天推荐它

  • 现在很多人学 AI 工程只停留在框架调用，这个项目更强调“把底层机制自己做一遍”。
  • 它既有课程感，也有仓库工程感，适合想系统建立知识树的工程师。
  • 如果你计划长期做 Agent、RAG 或模型应用，这类体系化仓库会比短平快教程更值钱。
