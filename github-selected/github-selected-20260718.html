1. alibaba/spring-ai-alibaba：把 Java 团队接入 Agent 时代的一整套工程底座

spring-ai-alibaba 是阿里开源的 Agentic AI 框架，定位非常清晰：让 Java 开发者基于熟悉的 Spring 生态，快速构建 Agent、Workflow 和 Multi-Agent 应用。它不是只做模型调用封装，而是把 Agent Framework、Graph Runtime、可视化 Admin 平台、Studio 调试能力以及 MCP 接入放到同一套体系里，更像一个面向生产场景的 AI 应用平台。

项目地址alibaba/spring-ai-alibaba
核心形态Agent Framework / Java
当前热度10.4k⭐ / 2.3k Fork（2026-07-18）
为什么今天选它：大量真实业务系统仍然跑在 Java 技术栈上，而这个项目提供了一条很清晰的路径——不用推倒重来，也能把 Agent、工作流和治理能力接进现有服务体系。
项目详细介绍

从官方 README 来看，spring-ai-alibaba 的分层设计很完整。上层是 Agent Framework，提供 SequentialAgent、ParallelAgent、RoutingAgent、LoopAgent 等多智能体编排模式；中层是 Graph Runtime，负责有状态、长生命周期工作流的编排、持久化与流式处理；外围再配套 Admin 和 Studio，用于可视化开发、调试、观测、评测以及 MCP 管理。它面向的不是玩具 Demo，而是“需要上线、需要观测、需要治理”的正式 AI 应用。

技术亮点
  • 内建多智能体编排模式，把串行、并行、路由和循环执行这些常见模式直接做成框架能力。
  • 提供 Context Engineering 与 Human in the Loop 机制，包括上下文压缩、上下文编辑、计划、动态工具选择、重试和人工介入。
  • Graph Runtime 支持条件路由、嵌套图、并行执行、状态管理，还能导出 PlantUML 与 Mermaid，方便团队交流与设计复盘。
  • 支持语音 Agent、多模态输入、A2A 协作、Nacos 集成，以及对多种模型提供方和 MCP 的统一接入。
使用场景

适合做企业内部智能助手、业务流程自动化 Agent、带审批或人工兜底的复杂工作流、多服务协同的智能系统，以及希望在现有 Java 微服务架构中平滑引入 AI 能力的团队。对技术成长来说，它特别值得拿来学习“AI 应用工程化”这件事：应用不只是调一个模型 API，而是要把编排、状态、治理、评测和可视化一起考虑进去。

2. opendatalab/MinerU：把 PDF、Office 和图片文档变成 LLM 可直接消费的结构化数据

MinerU 是一个非常实用的文档解析项目，核心价值是把 PDF、图片、DOCX、PPTX、XLSX 等复杂文档转换成 Markdown、JSON 等机器可读格式，方便后续做检索、抽取、知识库构建和 Agent 工作流。它不只是“抽个纯文本”，而是尽量保留文档结构、表格、公式、图片和阅读顺序，因此非常适合 RAG、研报解析、论文处理和企业文档数字化场景。

项目地址opendatalab/MinerU
核心形态Document Parsing Toolkit / Python
当前热度75.0k⭐ / 6.3k Fork（2026-07-18）
为什么今天选它：现在很多 AI 系统真正的瓶颈不在模型，而在输入数据质量。MinerU 直接卡在“复杂文档如何稳定变成高质量结构化语料”这个关键节点上，实践价值非常高。
项目详细介绍

官方 README 里给出的边界非常清楚：MinerU 支持 PDF、图片、DOCX、PPTX、XLSX 输入，能移除页眉页脚、脚注和页码，输出符合阅读顺序的文本结构，并抽取图片、表格、表格标题、脚注与公式。它还支持 Markdown、JSON 等多种输出格式，内置 CLI、FastAPI、Gradio WebUI 与多服务部署能力，既可以单机快速使用，也可以作为更大系统中的文档解析底座。

技术亮点
  • 覆盖 PDF、图片与 Office 文档，适合统一处理企业内常见非结构化资料来源。
  • 不仅提取文本，还保留标题、段落、列表、表格、公式和阅读顺序，更适合直接喂给检索与推理系统。
  • 支持扫描件 OCR，并宣称可覆盖 109 种语言，对跨语种资料处理比较友好。
  • 内置 CLI、API、WebUI 和路由能力，同时支持纯 CPU 环境，也支持 GPU / Apple Silicon 加速，部署弹性较好。
使用场景

适合做企业知识库入库、RAG 数据清洗、论文与研报解析、票据/合同/制度文档处理、多模态文档抽取，以及给 Agent 提供稳定的文档预处理能力。对技术成长来说，MinerU 很适合拿来补齐“数据入口工程”这一课：真正可用的 AI 系统，往往从高质量文档解析和结构化开始，而不是直接从模型对话框开始。
