1. alibaba/page-agent：把 AI 操作员直接放进网页里

page-agent 的方向非常讨巧：它不是在网页外面再套一层自动化脚本，而是把 Agent 直接嵌进页面本身，让网页原生具备理解自然语言、识别 DOM 结构并执行点击、输入、跳转等操作的能力。对 SaaS、管理后台、CRM、表单系统这类 Web 产品来说，这是一条非常轻量且落地感很强的 AI 增强路径。

项目名alibaba/page-agent
Star 数27,316 ⭐
简介JavaScript in-page GUI agent，支持用自然语言控制网页界面。
为什么今天选它：相比“浏览器自动化”老思路，它更像是在给现有网页产品加一层原生 AI 交互能力，接入门槛低，产品想象空间很大。
项目详细介绍

从官方 README 看，page-agent 主打 “One script gives any web page its own AI agent”。它不依赖浏览器扩展、Python 或 headless browser，而是直接在页面内通过 JavaScript 处理 DOM，并将自然语言命令转换为页面操作。项目支持自带模型，也支持接入主流大模型和本地模型；如果需要多页面任务，还能扩展到 Chrome Extension 和 MCP Server 形态。

技术亮点
  • 使用基于文本的 DOM 操作方式，而不是截图流和多模态识别，整体成本、延迟和工程复杂度更可控。
  • 接入非常轻量，支持脚本标签或 npm 包方式集成，适合在现有前端系统中快速验证 Agent 能力。
  • 支持自定义 LLM 和本地模型，便于企业在成本、隐私和供应商选择上保持灵活性。
  • 可选扩展 Chrome 插件与 MCP Server，从单页交互自然升级到跨页面和外部 Agent 控制场景。
使用场景

适合企业后台 Copilot、自然语言表单填写、客服/运营辅助、网页无障碍增强，以及 AI 驱动的网页助手。对技术成长来说，这个项目最值得学习的，不只是“怎么让 AI 点按钮”，而是“怎么把 Agent 以最小改造成本嵌入已有 Web 产品”。

2. DeusData/codebase-memory-mcp：给 Coding Agent 补上一层真正可复用的代码记忆

codebase-memory-mcp 解决的是另一个更偏工程底层、但对开发者极有价值的问题：AI 会写代码不难，难的是在大仓和多服务代码库中快速建立结构认知。这个项目把代码索引成持久化知识图谱，围绕函数、类、调用链、HTTP 路由和跨服务关系提供结构化查询，让 Agent 不用反复 file-by-file 地扫仓，而是直接在图谱上做理解和分析。

项目名DeusData/codebase-memory-mcp
Star 数33,316 ⭐
简介高性能代码智能 MCP Server，将代码仓索引为持久化知识图谱。
为什么今天选它：如果说大模型能力决定了 Agent 的上限，那代码知识图谱和上下文压缩能力，决定的就是 Agent 在真实工程环境里的下限。
项目详细介绍

官方 README 把它定义为 “High-performance code intelligence MCP server”。它基于 tree-sitter 做多语言 AST 分析，并通过 Hybrid LSP 增强语义解析，把仓库内容组织成一个可持续复用的知识图谱。项目强调单静态二进制、零依赖和本地运行，目标很明确：作为各类 Coding Agent 的基础设施层，为代码理解、检索、架构梳理和影响面分析提供高密度、低成本的上下文能力。

技术亮点
  • 支持 158 种语言解析，并对 Python、TypeScript、Go、Java、Rust 等主流语言提供更强的 Hybrid LSP 语义增强。
  • 把逐文件搜索模式升级为图谱查询、调用链追踪、影响面分析和跨服务关系探索，更适合复杂仓库。
  • 提供 15 个 MCP 工具，可覆盖搜索、trace、架构分析、死代码检查、HTTP 链路关联等研发高频问题。
  • 强调本地处理、零依赖和单二进制部署，适合对性能、隐私和稳定性都比较敏感的工程团队。
使用场景

适合大仓代码导航、Agent 辅助 Code Review、接口链路追踪、影响面评估、架构梳理，以及给 Claude Code、Codex、Cursor 这一类工具提供长期代码记忆。对技术成长来说，它值得重点关注，因为它代表了一个清晰趋势：未来真正好用的 Coding Agent，拼的不只是模型，而是背后有没有结构化、低成本、可复用的代码理解底座。
