1. anthropics/skills

Anthropic Agent Skills 公共仓库：用文件夹、说明、脚本和资源封装 Agent 的专项能力

TOP 1
🧩 Agent Skills🤖 AI Agent📦 Plugin Marketplace🧠 Claude Code📚 Skill Spec

项目详细介绍：anthropics/skills 是 Anthropic 公开的 Agent Skills 仓库。官方 README 将 Skill 定义为一组由指令、脚本和资源组成的文件夹，Claude 可按需动态加载这些内容，从而在特定任务上获得更稳定的表现。仓库中包含 document-skills、example-skills、skill-creator 以及 Agent Skills 规范链接，也支持作为 Claude Code Plugin marketplace 注册使用。它的价值不在于单个脚本有多复杂，而在于提出了一套“把经验、流程、工具调用和资源打包成能力单元”的组织方式。

技术亮点

  • 渐进式能力加载：Skill 以独立目录组织，Agent 只在需要时加载相关说明、脚本和资源，避免把所有上下文一次性塞给模型。
  • 结构化技能资产：把 prompt、操作步骤、辅助脚本和示例文件放在同一能力包内，利于复用、版本管理和团队协作。
  • 可扩展生态入口：仓库可注册为 Claude Code Plugin marketplace，并通过插件命令安装 document-skills 与 example-skills。
  • 规范化方向明确：项目把 Agent Skills 标准指向独立规范站点，说明它不只是样例库，也在推动 Agent 能力包的通用接口。
  • 适合工程团队沉淀经验：Code Review、日志分析、发布检查、文档生成等重复任务都可以被封装成 Skill，减少口口相传的流程损耗。

使用场景

  • 团队工程 SOP：把上线检查、故障复盘、接口评审等流程做成 Skill，让 Agent 按统一标准执行。
  • 文档与办公自动化：通过 document-skills 处理 DOCX、PPTX、XLSX、PDF 等格式转换与内容生成。
  • 领域知识注入：把业务术语、内部规范、常见坑和脚本放入能力包，使 Agent 更懂具体工作环境。
  • AI 工具产品设计：如果你正在做 Agent 平台，可以参考它的目录结构、元数据和按需加载思路。

为什么今天推荐它

  • 很多 AI 使用经验如果只停留在聊天记录里，很难复用；Skill 把这些经验变成可管理的工程资产。
  • 它提供了一个观察 Agent 产品演进的窗口：未来的竞争点会从“谁会写提示词”转向“谁能沉淀高质量能力库”。
  • 对技术成长而言，学习 Skill 的设计方式，有助于把自己的排障、开发、写作和分析流程系统化。

2. google-gemini/gemini-cli

Google 开源终端 AI Agent：把 Gemini 能力直接带入命令行开发环境

TOP 2
⌨️ CLI Agent🧠 Gemini🛠️ Developer Tooling🔌 MCP📄 Apache-2.0

项目详细介绍：Gemini CLI 是 Google 开源的命令行 AI Agent，官方定位是把 Gemini 的能力直接带到开发者终端中，用于编码、问题求解和任务管理。项目文档强调轻量、直接的交互路径，用户可以在终端中让模型理解项目、编辑文件、运行命令，并与现有开发工作流结合。相比只在 IDE 侧提供补全，它更像一个面向 shell、代码仓库和工程任务的通用 Agent 入口。

技术亮点

  • 终端原生体验：开发者无需离开 shell，就能让 AI 参与代码阅读、修改、解释和任务拆解。
  • 开源与可扩展：仓库公开，便于社区审计实现、贡献能力，并将其接入自定义工作流。
  • 长上下文模型入口：官方文档提到可访问 Gemini 2.5 Pro 与大上下文能力，适合处理跨文件理解和复杂任务。
  • 面向 Agent 工作流：CLI 形态天然适合结合 git、测试、构建、脚本和 MCP 工具，形成可自动执行的开发闭环。
  • 低迁移成本：对习惯命令行的工程师来说，Gemini CLI 更像增强现有开发环境，而不是替换 IDE 或编辑器。

使用场景

  • 代码库快速理解：让 AI 在终端中总结模块结构、解释调用链、定位可疑文件。
  • 小型改动与重构：把明确的修复、脚本调整、配置变更交给 CLI Agent 执行，再用 git diff 审查。
  • 排障辅助：结合日志、测试输出和命令执行结果，让 AI 协助分析失败原因。
  • 学习与练习：把它作为“命令行导师”，边运行、边解释、边修改，帮助理解真实项目工程流程。

为什么今天推荐它

  • CLI 是工程师最稳定的工作入口之一，AI Agent 进入终端意味着它可以更自然地连接代码、文件系统、命令和版本控制。
  • Gemini CLI 代表大厂把 AI Coding 从 IDE 插件扩展到通用开发工作流，值得关注其工具协议、权限边界和交互设计。
  • 对个人提升而言，它适合用来训练“AI 辅助开发的审查习惯”：让 Agent 做事，但自己用 diff、测试和日志把关。
