1. OpenHands/OpenHands

把编码 Agent、自动化任务和多种运行后端统一到一个可自托管的开发控制中心

TOP 1
⭐ 77,370 Stars 🍴 9,832 Forks 🐍 Python 📄 MIT 🏷️ coding agents / automation / self-hosting

项目介绍：OpenHands 已经不只是“一个 AI 写代码工具”，而更像是面向开发团队的 Agent 控制台。根据其 README，它主打 self-hosted developer control center，支持把 OpenHands、Claude Code、Codex、Gemini 等兼容 Agent 接到同一个前端，并在本地、Docker、VM、云端等多种后端环境之间切换。对想把 AI 开发能力真正接进日常工程体系的人来说，它很有代表性。

技术亮点

  • 统一 Agent 入口：不是只绑定单一模型或单一产品，而是把多类 coding agent 纳入同一个操作面，便于团队统一治理。
  • 多后端运行：既能在开发者本机跑，也能接 Docker、虚拟机或云端后端，更适合把个人实验延展到团队协作和持续任务。
  • 自动化工作流能力：支持和 Slack、GitHub、Linear 等工具集成，适合把日报、任务拆分、例行检查等动作做成自动化。
  • 自托管友好：项目明确支持 self-hosting，这对关注数据边界、权限控制和企业内落地的团队特别重要。

使用场景

  • 团队 AI 开发平台：统一接入多种 coding agent，降低“每个人各玩各的工具”带来的管理成本。
  • 自动化研发运营：把 issue 拆解、日报汇总、定时检查、代码协作等高频动作交给 Agent 持续执行。
  • 私有环境部署：适用于对代码、数据或研发流程有内网隔离要求的团队。
  • Agent 实验台：适合技术团队横向比较不同模型或不同 agent 的工作效果，而不是被某一家产品形态锁死。

为什么今天推荐它

  • 它代表了 AI 编码工具的下一阶段：不再只是“帮你写一段代码”，而是开始管理持续任务与团队协作流程。
  • 如果你已经在使用 Codex、Claude Code 或其他 Agent，这类平台型项目会决定你的长期工程效率上限。
  • 对有自托管、自动化、可审计需求的开发团队来说，它比单点插件更值得持续关注。

2. rasbt/LLMs-from-scratch

从文本数据、注意力机制到预训练与微调，手把手实现一个 GPT 风格大模型

TOP 2
⭐ 97,264 Stars 🍴 14,878 Forks 📓 Jupyter Notebook 📄 MIT 🏷️ llm learning / pytorch / from scratch

项目介绍：这个仓库是 Sebastian Raschka《Build a Large Language Model (From Scratch)》一书的官方代码仓库，但它本身也可以单独作为一套极强的学习材料。README 很明确：项目会一步一步带你实现 GPT-like LLM，覆盖数据处理、注意力机制、GPT 结构、预训练、分类微调、指令微调等完整链路。和“会调库”不同，它更适合建立真正的大模型底层心智。

技术亮点

  • 从零实现完整链路：不是只讲概念，而是让你亲手把 tokenizer、attention、GPT 模块、训练与微调代码真正写出来。
  • 章节化结构清晰：仓库按章节拆分，覆盖文本处理、注意力机制、预训练、分类微调、指令跟随微调等关键阶段。
  • PyTorch 友好：以 PyTorch 为主线，不依赖大而黑盒的 LLM 框架，适合理解模型内部运行机制。
  • 学习门槛可控：README 提到主线代码可在普通笔记本上运行，既适合入门，也适合做教学与课程配套。

使用场景

  • 系统学习 LLM：适合想摆脱“只会调 API”状态，真正理解大模型原理的工程师和学生。
  • 课程与训练营教材：很适合做公司内部分享、读书会、训练营或高校实验课的实战材料。
  • 研究前置准备：在做 LoRA、蒸馏、推理优化等进阶主题前，先建立扎实的基础实现能力。
  • 面试与技术成长：对想冲击算法、模型工程、AI Infra 方向的人，这类仓库是非常高性价比的成长路径。

为什么今天推荐它

  • 现在很多人会用大模型，但真正能讲清楚注意力、预训练和微调细节的人仍然不多，这个仓库正好补这个短板。
  • 它不是纯论文路线，而是把原理和代码结合得很紧，适合工程背景的人快速建立可落地理解。
  • 如果你想从“AI 使用者”进阶成“AI 构建者”，这是很值得长期投入的一套开源学习资产。
