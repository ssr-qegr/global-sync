NousResearch / hermes-agent

越用越聪明的自我改进型 AI Agent

TOP 1
个人智能体
Repository: NousResearch/hermes-agent方向：Self-improving Agent语言：Python

hermes-agent 的核心看点不是“又一个聊天 CLI”，而是把 Agent 的成长机制产品化。项目说明强调它会从经验中创建技能、在使用中改进技能，并支持在不同运行环境中使用；其文档也覆盖安装、CLI 使用、配置、提供商与模型切换等内容。对于长期使用 AI 助手的人来说，这类“把经验沉淀为可复用技能”的模式，比单次提示词更接近真正的个人工作系统。

技术亮点

  • 技能循环：把成功经验沉淀为 Skills，再在后续任务中复用和优化，降低重复劳动。
  • 跨模型接入：支持多种模型提供方和自定义 endpoint，适合做模型效果对比与成本控制。
  • 跨会话连续性：强调持久化记忆与会话管理，让 Agent 不只服务于一次任务。
  • 工程化入口：CLI、配置文件和文档体系完整，便于开发者把它当作本地工作台研究。

使用场景

  • 个人研发助手：沉淀常用排障、代码阅读、测试生成、发布检查等技能。
  • 知识工作流：把调研、摘要、计划、复盘拆成可长期复用的 Agent 流程。
  • 团队方法论验证：观察 Skills 如何管理、复用和演化，为团队内部 Agent 平台提供参考。
  • 模型供应商评估：在同一 Agent 工作流下切换不同模型，比较能力、速度和成本。

HKUDS / Vibe-Trading

自然语言驱动的金融研究与交易 Agent

TOP 2
领域智能体
Repository: HKUDS/Vibe-Trading方向：Finance Agent / MCP形态：CLI + Web + MCP Tools

Vibe-Trading 的价值在于把金融研究的多个环节做成工具闭环：市场数据获取、因子分析、期权分析、形态识别、回测、报告渲染、交易日志分析等都可以由 Agent 调用。项目 README 提到它提供 22 个 MCP tools，覆盖 backtest、factor_analysis、analyze_options、get_market_data、web_search、read_url、run_swarm 等能力，其中大部分工具无需 API Key 即可使用。它更像是一个“金融研究操作系统”的雏形，而不只是策略脚本集合。

技术亮点

  • MCP 工具化：把金融研究动作封装为 MCP 工具，适合接入 Claude Desktop、Cursor 等支持 MCP 的客户端。
  • 研究到回测闭环：从数据读取、因子计算、策略假设到 backtest 与报告输出，形成可验证链路。
  • Alpha Zoo：项目结构中包含多类 alpha 因子与运算符，适合学习量化研究的工程组织方式。
  • 低门槛实验：README 提到 22 个 MCP 工具中多数不需要 API Key，便于先做本地学习与原型验证。

使用场景

  • 量化学习：用自然语言驱动因子分析和回测，理解策略研究流程。
  • 研究助理：让 Agent 读取行情、文档与网页，生成初步分析结论和后续验证计划。
  • 交易复盘：结合交易日志分析、shadow strategy、shadow backtest 做策略归因。
  • MCP 案例学习：研究一个垂直领域如何把复杂任务拆成工具接口。
